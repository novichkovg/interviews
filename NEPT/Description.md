## Техническое задание

Нарисовать граф вычисления следующего алгоритма, оценить требуемые ресурсы для вычислений (количество вычислительных ядер, памяти), требуемую вычислительную производительность в TOPS и дать оценку по времени реализации-разработки алгоритма (до передачи его на тестирование) для запуска его работы на графическом ускорителе типа ARM Mali из состава RK3588. 

- На вход поступают данные с прибора, в режиме реального времени, потоком. Одно измеренное значение имеет диапазон значений 64 бит. Данные поступают с частотой 2 кГц. 
Необходимо непрерывно вычислять автокорреляционную функцию по мере поступления отсчетов, которая в отличие от классической АКФ должна представлять АКФ для каждой группы отсчетов, соответствующих определенной частоте. 

- Частотный диапазон выходных данных лежит от 1 Гц до 500 Гц, с шагом 1 Гц. На выходе алгоритма должна быть матрица АКФ по каждой строке которой записаны значения АКФ от минимальной до максимальной группы отсчетов, соответствующих частотам, Каждая вновь получаемая строка циклически сдвигается на одну строку вниз матрицы по мере поступления выходных данных. Формула для расчета 

$$АКФ = \int_{-T/2}^{T/2} S(t)  S(T-t) dt,$$

где:
- $T$ соответствует времени накопления для конкретной частоты, равное длине отсчетов,
- $t$ соответствует сдвигу по времени на ¼ длины по времени группы отсчетов входного сигнала соответствующего конкретной частоте. Частота равна $1/T$ . 
- $S(t)$ это группа отсчетов входного сигнала определенной длины для определенной частоты,

По итогам работы алгоритма в каждом элементе строки должна выводиться АКФ, соответствующая своей частоте. Размер выходной матрицы 500 (по частоте) х 2000 (по времени). 

Количество единовременно обрабатываемых АКФ на Mali 32.
